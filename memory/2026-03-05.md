### Evening Bug Fix (7:36 PM CST)
- **Bug**: Single image container layout issue where image only filled left half of container on desktop, and bottom left on iPhone. The swipe support's two‑half layout was misaligned.
- **Root cause**: Main preview container used `items-center` which horizontally centered the 200%‑width swipe container, leaving the left half offset. On iPhone, vertical centering may have been overridden by aspect‑ratio constraints.
- **Fix applied**:
  1. Changed main preview container from `items-center` to `items-start` (`InlineGalleryViewer.tsx` line 360) to align swipe container flush left.
  2. Updated adjacent image sizing from `w-auto h-auto max-w-full max-h-full` to `w-full h-full object-contain object-center` for consistent filling.
- **Result**: Image now fills the entire visible container area on both desktop and iPhone, while swipe transition still uses the right half for adjacent images only during active swiping.
