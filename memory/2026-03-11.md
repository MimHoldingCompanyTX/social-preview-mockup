### Upload Error Fix - Client-Side Image Compression (11:15 PM CDT)
- **Error confirmed:** "error response:service returned non-JSON error (413): Request Entity Too Large FUNCTION_PAYLOAD_TOO_LARGE."
- **Diagnosis:** Vercel serverless function rejecting files >4.5MB (413 error)
- **Solution implemented:** Client-side image compression before upload
- **Compression logic:**
  - Images >2MB automatically compressed using canvas API
  - Resized to max 1920px on longest side (preserves aspect ratio)
  - Quality: 85% for JPEG, 92% for PNG
  - Preserves original file extension (.jpg, .png, .webp)
  - Falls back to original if compression fails
- **Enhanced error messages:** Clear feedback when files still too large after compression
- **User experience:** Files show "(compressed)" in upload list when compression applied
- **Technical:** Added `compressImageFile` function to portal-content.tsx
- **Modified:** Upload loop to compress before size validation
- **Commit:** `f4391ab` "Add client-side image compression to fix Vercel 413 upload errors"
- **Expected result:** Most iPhone photos (typically 3-8MB) will be compressed to under 4MB
- **Limitations:** Non-image files (PDF, etc.) still subject to 4MB limit
